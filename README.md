Учебный проект по дисциплине МДК.05.02 Разработка кода информационных систем.
Тема: Макетирование каталогов сайта 
Выполнены **Задание 1** (прототип в Figma) и **Задание 2** (вёрстка каталога).

## Задание 1 — Прототип в Figma

**Ссылка на прототип:** [Открыть в Figma](https://www.figma.com/design/4dQWmkdCOr3I8RVYlE25kM/%D0%97%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5-10.03.26?node-id=0-1&t=v4LgHe1fscGG9mnX-1)

**Тематика:** книжный интернет-магазин.

### Что показано в прототипе

- **Десктоп** — полная структура страницы: sticky-шапка, hero-баннер, сайдбар с фильтрами, сетка карточек, пагинация.
- **Мобильный вид** — фильтры скрыты за кнопкой, сетка 2 колонки, кнопка «Показать ещё».
- **Карточка крупным планом** — все элементы карточки с подписями.

### Фильтры (5 штук)

| Фильтр | Тип элемента | Логика выбора |
|---|---|---|
| Жанр | Чекбоксы | Множественный выбор (ИЛИ) |
| Цена | Двойной ползунок | Диапазон от–до |
| Рейтинг | Радиокнопки | Один вариант — минимальная оценка |
| Язык | Чекбоксы | Множественный выбор |
| Год издания | Чекбоксы | Множественный выбор по периодам |

### Расположение элементов

- **Sticky header** — всегда доступен поиск и корзина.
- **Hero-баннер** — заголовок категории, задаёт контекст.
- **Sidebar (260 px, sticky)** — фильтры слева; стандарт e-commerce, снижает когнитивную нагрузку.
- **Активные теги** над сеткой — моментальная обратная связь о применённых фильтрах.
- **Adaptive grid** — карточки сами перераспределяются при изменении ширины.

### Тип фильтрации

- **Клиентская** (JavaScript) — мгновенный отклик при изменении чекбоксов и ползунка, debounce 300 мс.
- **Серверная** при первой загрузке — параметры передаются через URL (`?genre=fiction&price_max=2000`), что обеспечивает SEO и возможность поделиться ссылкой.

### Адаптив

| Ширина | Поведение |
|---|---|
| > 900 px | Двухколоночный layout: sidebar + основной контент |
| ≤ 900 px | Sidebar скрыт; появляется кнопка «Фильтры» — открывает блок сверху |
| ≤ 600 px | Поиск на всю ширину, кнопки шапки скрыты, сетка 2 колонки |

### Подход

Прототип выполнен в стиле **low-fidelity**: серые блоки вместо изображений, текстовые строки заменены прямоугольниками, цветовая схема не применяется. Это позволяет сосредоточиться на структуре и расположении элементов, не отвлекаясь на визуальный дизайн. Типы элементов управления подписаны прямо на макете: `[чекбоксы]`, `[ползунок]`, `[радиокнопки]`.

## Задание 2 — Вёрстка

### Технологии

- Чистый **HTML5 / CSS3 / Vanilla JS** — без фреймворков.
- Семантические теги: `<header>`, `<aside>`, `<main>`, `<article>`, `<footer>`.
- Шрифты: **Playfair Display** (заголовки) + **DM Sans** (текст) — Google Fonts.

### Что реализовано

- ✅ 8 карточек книг в адаптивной сетке.
- ✅ Каждая карточка: цветная обложка-плейсхолдер, жанр, название, автор, рейтинг, цена (со скидкой или без), кнопка лайка ♡/♥, поле для комментария.
- ✅ Бейджи «Новинка» и «−XX%» на карточках.
- ✅ 5 фильтров с рабочим UI (чекбоксы, двойной ползунок, радиокнопки).
- ✅ Теги активных фильтров с удалением по клику.
- ✅ Кнопка сброса всех фильтров.
- ✅ Сортировка (select) и переключатель вида.
- ✅ Пагинация + кнопка «Показать ещё».
- ✅ Скругления, тени, hover-анимации, staggered fadeUp при загрузке.
- ✅ Полная адаптивность (mobile-first breakpoints).

### Как открыть

Скачайте все файлы в одну папку и откройте `index.html` в любом современном браузере — никакого сервера не нужно.

---

### Дизайн-решения

- **Цветовая палитра:** тёмно-коричневый (`#1a1208`) + янтарный (`#c8861e`) — ассоциация с кожаными переплётами и старой бумагой.
- **Типографика:** засечный шрифт Playfair Display создаёт «книжную» атмосферу.
- **CSS-переменные** (`--ink`, `--amber`, `--parchment` и др.) — единый источник истины для всей палитры.
